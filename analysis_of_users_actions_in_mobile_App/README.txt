Анлиз действий пользователй в мобильном приложении для покупки продуктов питания

Цели исследования: 

Источник данных: имеется датасет лога посетителей мобильного приложения со следующими полями: 
    
        - EventName - название события; 
        - DeviceIDHash - уникальный индентификатор пользователя;
        - EventTimestamp - время события;
        - ExpID - номер эксперимента, где 246 и 247 - контрольные группы, а 248 - экспериментальная; 
1.На первом этапе необходимо изучить воронку продаж: 
        - изучить цепочку событий до покупки; 
        - доли пользователей дошедших до покупки и не дошедших, установить шаги на которых они отсеялись;
        
2.На втором этапе необходимо оценить результаты внедрения новых шрифтов в приложении. Для этого отбрано две группы пользователй: две контрольные - 246 и 247 со старыми шрифтами, и одна экспериментальная с новыми шрифтами - 248. После проведения A/A/B-теста можно сделать вывод об успешности внедерения шрифтов в приложение

План исследования: 
1. произвести предобработку данных:
- привести наименвания столбцов в датасетах к одному стилю;
- проверить данные на пропуски и провести их обработку;
- проверить данные на неявные дубликаты;
- проверить корректность формата данных датасета и привести их к необходимому формату;
- добавить необходимые столбцы к датасетам для проведения анализа;
2. Произвести анализ представленных данных:
- оценить количество событий в датасете лога;
- оценить количество пользователей в логе;
- рассчитать среднее количество событий на одного пользователя;
- оценить временной интервал представленного датасета, определить полноту временных данных и произвести фильтрацию данных;
- оценить количество потерянных данных после фильрации; 
- проверить наличие пользователей из всех эксперементальных групп;
4. Изучить воронку событий:
- определить список уникальных событий и определить их частоту;
- рассчитать количество пользователей, совершавших одно из событий.
- рассчитать долю пользователей, совершавших хотябы одно из событий;
- сделать предположение о порядке последовательности событий.
- рассчитать конверсию пользователей в отношении шагов воронки;
- определить шаг воронки, на котором теряется больше всего пользователей; 
- определить долю пользователей, которая доходит от первого события до оплаты;
5. Изучить результаты эксперимента
- оценить размерность каждой экспериментальной группы;
- проверить есть ли статистическая разницу между выборками 246 и 247;
- оценть количество пользователей, совершивших самое популярное событие;
- оценить долю пользователей, которые совершили наиболее популярное событие;
- проверить будут ли различия между этими группами статистически достоверны;
- произвести проверку аналогичную проверку для остальных событий;
- произвести аналогичную проверку с экспериментальной группой 248 и призвести сравнение результатов с объединенной контрольной группой;
